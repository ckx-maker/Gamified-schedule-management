# Gamified-schedule-management

> 把游戏化机制搬进日程管理——让"完成任务"像"升级"一样上瘾。
一个用单 HTML 文件实现的极简周历任务管理器。不依赖任何框架、不需要构建步骤，打开即用。灵感来源于一款让人连续沉迷多日的游戏——我拆解了它让我上瘾的底层逻辑，然后把这些机制重新设计成了日常时间管理工具。

📱 **在线体验**：下载 `Gameified-schedule- management.html`，用浏览器打开即可。建议用手机或 Chrome 开发者工具的移动端模式获得最佳体验。

## 为什么做这个

### 起源

我玩了一款游戏。游戏的核心是：通过任务管理界面，一步一步达成不同阶段的成就。它让我连续玩了很多天。

我开始思考：**真正让我上瘾的到底是什么？**

拆解下来，是四个机制在起作用：

| 机制 | 作用 |
|------|------|
| **即时反馈** | 每完成一步都立即给予反馈，不拖延满足感 |
| **成长可视化** | 整个成长过程看得见，不是抽象的"我变好了" |
| **降低决策成本** | 不需要思考下一步做什么，系统已经安排好了 |
| **持续激励** | 每天都会产生一点新的期待 |

于是问题来了：

> 如果把这些机制放到时间管理里，会不会比传统 Todo 更容易坚持？

### 过程

这个项目完全通过 **Vibe Coding**（AI 辅助编程）完成。从构想到成品，经历了多轮迭代：

1. **第零步 · 逆向拆解** — 截图游戏界面，用 AI 工具分析功能模块，提炼出可复刻的核心交互
2. **第一版 · 功能验证** — 让 AI 直接复刻游戏日程管理界面，验证"点击派遣任务"的交互是否成立
3. **第二版 · 交互重构** — 弃用拖拽方案，改为"点一下任务库 → 点一下日程"的直接派遣；加入长按编辑、进度次数等机制
4. **第三版 · 网页部署** — 部署到 Netlify，但发现整体 UI 偏网页化，不符合手机端使用习惯；结算界面也未达预期
5. **最终版 · 重新定调** — 根据[产品需求文档](#)重新定义视觉方向，确定三色系统、iOS 极简风、玻璃质感任务库，完成最终实现

> 关键决策：每一版迭代都不是推倒重来，而是基于上一版的用户反馈做减法——砍掉不自然的交互，强化有效的反馈。

## 核心功能

### 一周日程管理

- **周历视图**：一屏展示整周安排，选中某天即可编辑当日日程
- **任务派遣**：点击任务库中的卡片，自动派遣到当前选中的日期；再点一次取消
- **进度系统**：每个任务可设定目标次数（如"健身 5 次"），同一天可多次派遣同一任务
- **多周导航**：支持前后翻周，但不能回到过去（时间只能向前）

### 四类任务体系

| 类别 | 金钱 | 进度 | 说明 |
|------|------|------|------|
| 💼 工作 Work | 💰 赚钱 | ✅ 有 | 完成获得金币奖励 |
| 📚 培训 Growth | 💸 花钱 | ✅ 有 | 提升能力但消耗金币 |
| 🎯 机会 Chance | — | ✅ 有 | 不涉及金钱，纯成长 |
| 🌙 休息 Rest | — | ❌ 无 | 纯放松，减压恢复 |

### 六维属性系统

每个任务会影响角色的六项人生属性：

- **压力 Stress** — 越低越好
- **财富 Finance** — 工作赚钱，培训花钱
- **表达 Voice** — 沟通与表现力
- **活力 Vigor** — 精力与体能
- **格局 Vision** — 认知与眼界
- **外形 Look** — 外在形象

### 结算系统

- **今日任务页**：逐个点击完成任务，实时显示进度条
- **结算动画**：每个任务结算时展示对应背景图、属性变化、金币变动
- **评级机制**：根据完成率给出 S/A/B/C/D 评级，最后一个任务结算时弹出评级徽章
- **自动顺延**：22:00 后未结算的任务自动留到第二天

### 统计归档

- 按时间顺序记录每一次结算
- 汇总显示：已归档数、金币净额、连续天数
- 支持删除单条记录

## 设计哲学

### 三色系统

整个应用只用三个颜色，**不使用蓝色**：

```
背景    #f3f1ec  米白
文字    #26241f  墨黑
辅助    #8a857b  暖灰
```

> 约束即自由。三色系统让整个应用保持视觉统一，任何元素都不会显得突兀。

### iOS 极简风

- 主打"轻"感，所有设计服务于"无负担使用"
- 圆角卡片 + 柔和阴影，保持原生 iOS 质感
- 手机全屏浏览，不需要上下滑动
- 玻璃质感的任务库（liquid glass 风格），上下滚动有立体感

### 交互原则

- **低决策成本**：任务已经排在任务库里，点一下就派遣，不用想"今天该做什么"
- **即时反馈**：派遣有动画，完成有勾选，结算有评级
- **零摩擦操作**：取消任务 = 再点一次；编辑任务 = 长按；不需要进二级菜单

## 技术实现

### 技术栈

- **纯 HTML/CSS/JS** — 单文件，零依赖，零构建
- **CSS Variables** — 统一管理三色系统和设计 token
- **Flexbox + Absolute Positioning** — 手机端适配布局
- **localStorage-ready** — 数据结构已设计好，可扩展持久化

## Vibe Coding 实践笔记

这个项目是 Vibe Coding 的一个实践案例——从产品构想到最终实现，全程由 AI 辅助完成。以下是一些经验总结：

### 什么有效

- **截图驱动开发**：把参考界面截图发给 AI，让它分析功能模块，比纯文字描述高效十倍
- **逐步迭代**：不要一次性要求 AI 完成所有功能，每一轮只聚焦一个核心问题
- **逆向工程思维**：先拆解"为什么有效"，再决定"怎么做"

### 什么需要警惕

- **AI 会给你想要的，但不一定是对的**：第一版完全复刻游戏 UI 看起来很酷，但实际使用时发现横屏不符合手机习惯
- **视觉方向要人定**：AI 可以实现任何风格，但"三色系统""不用蓝色""iOS 极简"这些决策必须由人来拍板
- **多轮反馈是必须的**：没有第三版部署后的"不对劲"感受，就没有最终版的重新定调
